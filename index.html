Data extracting and preprocessing details

Data extraction involves obtaining raw chess game data using the Chess.com API. The API provides a list of monthly archive URLs for a player's games. The script iterates through these URLs to fetch all game data for the given username. Each archive contains games with information like player ratings, results, PGN (Portable Game Notation), and metadata (e.g., start/end time, ECO code for openings). All games are stored in a JSON file for further preprocessing.

Preprocessing converts the raw data into a clean and structured format suitable for analysis. Determines whether the user played as white or black. Captures the opponent’s color and ratings. Removes the opponents username in order to ensure their privacy. Extracts the user's game result (win, loss, or draw). The script extracts the ECO code from the game metadata. Converts it into a readable format (e.g., "Ruy Lopez" instead of "C60"). Calculates the total duration of the game using start and end times from the PGN. Converts duration into a readable format (MM:SS). Counts the total number of moves in the game using regex on the PGN string. Checks the PGN to determine whether the user or the opponent castled (kingside or queenside). Converts the game's end time (epoch format) into a human-readable datetime. Data is filtered thus only "rapid" games are included.

P-Value Hypothesis Testing Details

I analyzed whether my choice of chess opening significantly affects my win rate using ANOVA. First, I create a numeric "win" column where wins are 1, draws are 0.5, and losses are 0. Then, I filter out openings that I have played less than 10 times to ensure I have enough data for meaningful analysis. For the remaining openings, I group my games by opening and calculate the win rates for each. Using an ANOVA test, I compute the F-statistic and p-value to determine if there are significant differences in win rates between openings.

Null Hypothesis (H0): There is no significant difference in win rates between opening choices

Alternative Hypothesis (Ha): Different opening choices lead to significantly different win rates

P-Value: 0.11

Conclusion: With p > 0.05, we fail to reject the null hypothesis. Different opening choices do not show statistically significant differences in win rates.

Performance Analysis

Win Rate by Time of the Day

There are winrate differences depending on time of day.

P-Value Hypothesis Testing Details

I analyzed the impact of the time of day on win rate by first converting the 'time' column to a datetime format and extracting the hour of the day. Then I categorized the hours into four parts of the day: "Morning" (0-9), "Afternoon" (9-17), "Evening" (17-21), and "Night" (21-24). I also created a new column for win rate, where a win is marked as 1 and a loss as 0. Then I calculated the average win rate for each time of day and print the results. Finally I performed a two-sample t-test to compare win rates between the morning/afternoon group (hours ≤ 17) and the evening/night group (hours > 17).

Null Hypothesis (H0): There is no significant difference in win rates between playing in different times of day

Alternative Hypothesis (Ha): Playing in different times of day leads to significantly different win rates

P-Value: 0.86

Conclusion: With p > 0.05, we fail to reject the null hypothesis. Playing in different times of day does not show statistically significant differences in win rates.
